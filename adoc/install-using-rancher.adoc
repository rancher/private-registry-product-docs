[#pr-install-rancher]
== Installation using the {ranchera} UI

:revdate: 2026-05-05
:page-revdate: {revdate}

To install {productname} using the {ranchera} UI, you must meet the following requirements and follow the steps below.

[#pr-install-rancher-requirements]
=== What requirements do I need to meet?

* An environment with a running supported {kube} cluster(s) and {ranchera} deployed.
Refer to link:https://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/requirements/requirements.html[{ranchera} installation requirements] for more details.
* {sregistry} credentials for an account with a {productname} subscription (a username and a password/token).
Refer to link:https://documentation.suse.com/en-us/cloudnative/suse-private-registry/html/private-registry/pr-deployment.html#pr-deployment-kube-secrets[Obtaining {kube} secrets from {scca}] for more details.


[#pr-install-rancher-procedure]
=== What are the steps to install {productname} using the {ranchera} UI?

.*STEP 1*: Tell {ranchera} where the {productname} repository is located to look for the installation chart.
. Log in to {ranchera}.
. Click the three-line menu (☰) in the top-left corner, select menu:Cluster Management[], and click your cluster's name, usually `local`.
. From the left-hand menu, select menu:Apps[Repositories].
. Click the btn:[Create] button in the top right and complete the form that opens:
  .. *Target*: Select btn:[OCI Repository].
  .. *Name*: Enter a name for the repository, such as  `SUSE Private Registry`.
  .. *Description*: Optionally, add a description of the repository.
  .. *OCI Repository Host URL*: Enter \`oci://registry.suse.com/private-registry/private-registry-helm`.
  .. *Authentication*: Change to `Create an HTTP Basic Auth Secret` and enter the user name and password of the registry's credentials.
. Confirm with btn:[Create].

.Adding a {productname} repository
image::spr-rancher-add-repo.png[A screenshot showing how to add a {productname} repository to {ranchera}, width=100%]

.*STEP 2*: Create a secret to access the images in the \`registry.suse.com`.
. Click the three-line menu (☰) in the top-left corner and select menu:Cluster Management[].
. Switch to the cluster to which you want to add the secret and click btn:[Explore].
. To navigate to secrets management, select menu:Storage[Secrets] and click btn:[Create] in the top right.
. Select the btn:[HTTP Basic Auth] secret and then the `private-registry` namespace.
. Enter `suse-registry` as the name for the secret.
. Fill the `username` and the `password` fields with the {suse} credentials obtained in xref:pr-install-rancher-requirements[].

.Adding {productname} secrets
image::spr-rancher-add-secrets.png[A screenshot showing how to add {productname} secrets to {ranchera}, width=100%]

.*STEP 3*: Install the {helm} chart.
. From the main left menu, select menu:Apps[Charts].
. Enter the name of the assigned {productname} repository into the search box. For example, `SUSE Private Registry`.
If it does not appear in the list, click btn:[Refresh all repositories].
. Click the chart and view the `README.md`.
. Optionally, you can customize the installation values.
Either click through the sections on the left side of the `Edit Options` panel to view all the values you can configure, edit the values directly in the chart's YAML file.
. In the top right corner, click btn:[Install this version].
